# 03 — v1.0.0 リリース判定 checklist

このファイルは ecc-method を **試行錯誤期 (v0.x)** から **公開・配布期 (v1.0+)** へ昇格させる際の判定 checklist。
原典は `45_runbook/runbooks/RB-008-branch-strategy-and-semver.md` §リリース判定基準。本ファイルは進捗追跡用の独立チェックリスト。

## 0. v1.0.0 の意味 (RB-008 §semver 運用 より)

| 版 | 意味 |
|---|---|
| v0.x.y | 試行錯誤期、API 不安定。破壊的変更を許容。 |
| v1.0.0 | 公開・配布期。API 安定保証。破壊的変更は v2.0 へ昇格。 |

v1.0.0 を切ると、章構成・原則・採番・Runbook ID 等の **互換性保証** が発生する。

## 1. 判定基準 (すべて満たして昇格)

### 1.1 構造の安定

- [ ] 章採番ポリシー確定: 5 刻みを単一規律として明示 (HW-D Session 3, 2026-06-24 完了)
- [ ] トップレベル採番に 5 刻み破りが存在しない
- [ ] 全章末尾に `## 出典` `## 不確実性` セクションがある
- [ ] CHANGELOG.md が整備されている (Keep a Changelog 形式、v0.x 期の変更履歴を含む)

### 1.2 内容の汎用化

- [ ] 全章で個人名 / 組織名 / 案件固有プロダクト名ゼロ
- [ ] 全章で絶対パスゼロ (相対パスのみ)
- [ ] 30_sdd-phase / 35_tdd-phase の汎用化精査が完了 (HW-F)
- [ ] 案件依存 / 実機未検証 / 出典なしの箇所がすべて解消または `## 不確実性` に記載

### 1.3 検証

- [ ] 主要 URL の生死確認が直近 30 日以内 (`85_frontier/` の検証ステータス表)
- [ ] 別ユーザー / 別案件で 3 ステップ導入が成功している (再現性確認、最低 1 件)
- [ ] 機械検査 (個人/組織情報・絶対パス・出典なし主張) が pass

### 1.4 配布準備

- [ ] LICENSE が明示されている (現状 MIT)
- [ ] README が利用者視点で書かれている (開発者向けではない)
- [ ] `.gitignore` で `.session-state/` が main 配布から除外されている
- [ ] `99_distribution/01_how-to-redistribute.md` が最新
- [ ] `.template-agents/ecc-orchestrator.md` が develop の最新内容と一致 (起動時 §1 のディレクトリ参照、終了時 §6 の 4 要素規律など、Runbook と乖離していないこと)
- [ ] `.template-agents/ecc-orchestrator.md` の起動時/終了時 prompt と RB-006/RB-007/RB-009 本文が grep で照合し一致
- [ ] `.template-claude/CLAUDE.md` の §5 クローズ規律書式が RB-006 §[6] と完全一致 (`完了:` / `次回継続:` / 定型文 / `# ⚠️ このセッションはクローズ` の 4 要素 + `#` 見出しマーキング)
- [ ] `.template-claude/CLAUDE.md` 内に絶対パス・個人ホーム・組織内パスが含まれない (`<ECC_METHOD_ROOT>` プレースホルダのみ)
- [ ] `.template-claude/README.md` の採用手順 3 ステップが現行ディレクトリ構成と整合

> 配布版整合の根拠 (Session 5, 2026-06-24): main `v0.1.0` の `.template-agents/ecc-orchestrator.md` は旧 `.handover/` 参照 + 終了時規律「1 行含める」のまま固定されており、develop の修正 (`.session-state/` 移行 / 4 要素クローズ末尾固定) が未反映。v1.0.0 切り時に develop → main の merge で必ず反映されることを保証する。

## 2. 昇格手順 (RB-008 §リリース手順 と整合)

```
[1]  develop で本 checklist を確認、すべて [x] になっていること
[2]  CHANGELOG.md の [Unreleased] を [v1.0.0] - YYYY-MM-DD に確定
[3]  develop で commit "chore: prepare release v1.0.0"
[4]  git checkout main
[5]  git merge develop --no-ff -m "release: v1.0.0"
[6]  .gitignore で .session-state/ が除外されていることを確認
[7]  git tag -a v1.0.0 -m "v1.0.0 - 公開・配布期へ昇格"
[8]  git push origin main
[9]  git push origin v1.0.0
[10] GitHub Releases UI で release note を作成 (CHANGELOG から転記)
[11] develop で次バージョン (Unreleased) の準備を再開
```

## 3. 昇格後の規律 (RB-008 §公開期 と整合)

- 破壊的変更は v2.0 へ昇格 (Major bump)
- 利用者の既存 `.session-state/` 互換性を保つ (schema 変更は major bump)
- deprecate は 1 minor 期間予告 → 次 major で削除
