# 04 — Uncertainty Language (未検証・推測の明示語彙)

不確実な主張を「断定」と区別するための語彙規約。ハルシネーションの抑止と読み手の判断材料の確保が目的。

## 1. 4 段階の確実性ラベル

| ラベル | 意味 | 使用語彙 |
|---|---|---|
| **検証済み** | L1 出典あり、または当パッケージで再現確認済 | 「〜である」「〜する」(断定) |
| **公式情報依拠** | L1 出典あり、ただし当方では未再現 | 「公式仕様によれば〜」「ドキュメントは〜と記載している」 |
| **推測** | L2 / 状況証拠ベース、断定不可 | 「〜と推測される」「〜と思われる」「〜の可能性が高い」 |
| **未検証** | 出典がない / L3 のみ / 仮置き | 「未検証」と明示 |

## 2. 表現テンプレート

### 2.1 検証済み (L1)

```
Claude Code の TodoWrite ツールは進行中タスクを 1 件のみ許容する
(L1: code.claude.com/docs/todowrite, retrieved 2026-06-23)。
```

### 2.2 公式情報依拠 (L1 だが未再現)

```
公式ドキュメントによれば prompt cache の TTL は既定で 5 分である
(L1: docs.claude.com/en/docs/build-with-claude/prompt-caching, retrieved 2026-06-23)。
本パッケージでは未再現。
```

### 2.3 推測 (L2 / 間接証拠)

```
multi-agent research system の効率化要因は parallel tool use と推測される
(L2: anthropic.com/engineering/multi-agent-research-system, retrieved 2026-06-23)。
```

### 2.4 未検証

```
**未検証**: GPT-5.5 の context window は内部資料で 400k と記載があるが、
本パッケージでは出典が確保できなかった。再検証は次回 industry radar で実施する。
```

## 3. 「## 不確実性」セクションの書き方

各 `.md` の末尾には「## 不確実性」セクションを置く。本文中の未検証・推測項目をここに集約する。

```markdown
## 不確実性

- (未検証) 〇〇の P95 値は社内ベンチで取得した参考値。公式値は未公表。
- (推測) 〇〇の挙動は L2 出典に基づく推測であり、本パッケージで再現確認していない。
- (前提) 〇〇は 2026-06-23 時点の挙動。四半期ごとの再検証対象。
```

各エントリの先頭に **(未検証) / (推測) / (前提)** のいずれかをラベル付けする。

## 4. 推測語彙の許容と非許容

### 4.1 許容 (推測の明示として使用可)

- 「〜と推測される」
- 「〜の可能性が高い」
- 「〜と仮定する」
- 「〜と見られる」
- 「公式には未確認」

### 4.2 非許容 (曖昧の温存・責任回避)

- 「〜だと思います」 (一人称の主観)
- 「〜らしい」 (出典不明の伝聞)
- 「〜のはず」 (希望的観測)
- 「おそらく〜」 (定量化なし)
- 「一般に〜と言われている」 (L3)

## 5. 数値の不確実性

数値主張は以下のいずれかを満たすこと。

1. L1 出典の値 (URL + retrieved_at 必須)
2. 自前計測値 (計測条件・サンプル数・測定日を併記)
3. 推測値 (「概算」「目安」と明示し、根拠を併記)

```
未許容: 「処理は約 1 秒で終わる」(出典なし)
許容:   「処理は約 1 秒で終わる (社内ベンチ n=10, P50, 2026-06-20 時点)」
許容:   「公式仕様では P95 1.2 秒 (L1: ..., retrieved 2026-06-23)」
```

## 6. 「分からない」と書く勇気

調査して分からなかった項目は「分からなかった」と明示する。捏造より遥かに価値がある。

```
**未検証**: prompt cache の最大エントリ数は本調査では確認できなかった。
公式 docs および公式 blog を 2026-06-23 時点で検索したが言及を発見できず。
```

## 7. 自動検査

`60_quality-gates/05_uncertainty-disclosure.md` で以下を確認する:

- 各 `.md` に「## 不確実性」セクションがあるか
- 数値主張の出典付帯率
- 「らしい / だと思います / のはず / おそらく」のヒット (warn)
- 「一般に〜と言われている」「と聞いた」のヒット (block)
