# 01 — Voice (第三者中立・コンサル文体)

本パッケージおよび本パッケージを適用する案件で生成される全ドキュメント (PRD / requirements / design / tasks / Runbook / review レポート等) に適用する文体規範。

## 1. 基本方針

- **第三者中立**: 書き手・読み手・対象システムを区別し、語り手の主観を持ち込まない。
- **コンサルティング文体**: 事実 → 含意 → 推奨アクションの順で書く。感情語・自賛語を排する。
- **出典駆動**: 主張には L1 出典を付帯する ([03_citation-style.md](./03_citation-style.md))。
- **不確実性開示**: 未検証・推測は明示する ([04_uncertainty-language.md](./04_uncertainty-language.md))。

## 2. 文体ルール

### 2.1 一人称・二人称を抑制する

- 一人称 (「私」「我々」「弊社」) は原則使わない。やむを得ない場合のみ「本パッケージ」「本ドキュメント」と表記。
- 二人称 (「あなた」) は手順書 (Runbook) と一次対応 (06_user-care/) に限定する。
- 解説・分析・設計の本文は無人称で書く。

### 2.2 文末は断定または条件付き断定に統一

- OK: 「〜である」「〜する」「〜の場合は〜する」
- 避ける: 「〜と思います」「〜だと感じます」「〜してみましょう」「〜できるはず」

### 2.3 主観形容詞を使わない

「素晴らしい」「強力な」「画期的な」のような主観形容詞は使わず、**測定可能な事実**で代替する。

| 主観形容詞 | 中立化 |
|---|---|
| 「強力な検索機能」 | 「N 件のドキュメントを 200ms 以内で返す検索機能」 |
| 「直感的な UI」 | 「初回操作完了まで平均 3 ステップの UI (社内テスト n=12)」 |
| 「高速な処理」 | 「1,000 件あたり 1.2 秒の処理 (検証環境)」 |

### 2.4 自賛・卑下を排する

- 自賛 (自己評価語の使用) は避ける。判断手続きは [02_avoidance-patterns.md](./02_avoidance-patterns.md) §2.1 を参照。
- 卑下表現も同様に避ける。事実だけ書く。
- 擬人化・関係性表現は避ける。判断手続きは [02_avoidance-patterns.md](./02_avoidance-patterns.md) §2.2 を参照。

個別語の列挙はしない (§2.4 の判断は文脈依存。同じ語でも文脈により正常使用される)。

### 2.5 個人名・組織名・固有プロダクト名を出さない

- 本パッケージ内では Anthropic / OpenAI 等の **公式仕様の出典としての社名** は許容するが、それ以外は避ける。
- 案件側ドキュメントでは顧客名・社員名・特定取引先名を含めない。例示が必要な場合は `<company-x>` `<vendor-y>` のように記号で抽象化する。

### 2.6 含意 → 推奨アクションの 2 段で締める

事実列挙だけで終わらせない。各セクションは以下の構造を取る:

```
[事実]   …である / …が観測された (出典)
[含意]   これは … を意味する
[推奨]   よって … する / … を実行する
```

### 2.7 文長

- 1 文 80 字以内を目安。
- 1 段落 5 文以内。
- 階層化が必要なら箇条書きまたはテーブルに切り出す。

### 2.8 受動態より能動態

- OK: 「Orchestrator が委任契約を発行する」
- 避ける: 「委任契約が発行される」

ただしユーザー操作を主語にしない場合 (システム動作の説明) は受動態でも可。

## 3. コンサル文体の構造テンプレート

```
## 概要
- 1 行で結論

## 現状
- 観測された事実 (出典付き)

## 含意
- なぜそれが問題か / 機会か

## 推奨アクション
- 何を / 誰が / いつまでに

## リスクと前提
- 採用しない選択肢
- 不確実性

## 出典
- L1 出典のリスト
```

## 4. 例外

- README / overview の冒頭の 1 行サマリは平易な表現を許容する。
- 案件のユーザー向けマニュアルは平易体に切り替えてよい。ただし禁止語は適用する。
